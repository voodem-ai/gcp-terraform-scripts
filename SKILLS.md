# Skill File: Project Requirements & Competencies

To successfully execute the `gcp-terraform-scripts` project and guide the AI assistant, the following skills and knowledge areas are required:

## 1. Cloud Infrastructure (Google Cloud Platform)
- **VPC Networking:** Understanding of subnets, secondary IP ranges (alias IPs), Cloud NAT, and Cloud Router.
- **Google Kubernetes Engine (GKE):** Deep understanding of VPC-native clusters, Private Clusters, Node Pools, and Control Plane security.
- **Identity & Access Management (IAM):** Knowledge of GKE Workload Identity and principle of least privilege for service accounts.

## 2. Infrastructure as Code (Terraform)
- **HCL (HashiCorp Configuration Language):** Ability to read, review, and debug Terraform code generated by the AI.
- **State Management:** Understanding of local vs. remote state (e.g., using Google Cloud Storage for remote state).
- **GCP Provider:** Familiarity with the `google` and `google-beta` Terraform providers and their documentation.

## 3. Kubernetes Architecture & Networking
- **Workload Management:** Understanding Deployments, Pods, and scaling.
- **Service Exposure:** Differentiating between `ClusterIP` (internal routing for AI agents), `NodePort`, and `LoadBalancer`.
- **Ingress Controllers:** Configuring GKE Ingress to provision GCP HTTP(S) Load Balancers securely.
- **Network Policies:** Optional but recommended skill to write rules that explicitly prevent the Frontend pods from accessing backend AI agents on unauthorized ports.

## 4. AI-Assisted Engineering (Prompting)
- **Context Providing:** Ability to give clear architectural constraints (e.g., "Generate a private GKE cluster in Terraform, ensuring nodes have no public IPs").
- **Iterative Refinement:** Skill in reviewing AI-generated code, spotting security flaws (like a missing firewall rule or a public IP assignment), and prompting the AI to correct it.
